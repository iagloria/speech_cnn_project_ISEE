## 摘要

本文设计并实现了一个基于 **梅尔频谱图 (Mel Spectrogram) + 二维卷积神经网络 (2D CNN)** 的端到端语音命令识别系统。系统首先通过短时傅里叶变换 (STFT) 与梅尔滤波器组将一维语音信号转化为二维时频表征，然后将频谱图作为"图像"输入 CNN 进行分类。项目从 20 类英文命令词出发（V1，准确率 96.49%），逐步扩展到 40 类中英双语识别（V3，英文 94.5%，中文 82.9%），并通过类别加权损失函数解决了中英文数据极度不均衡问题。实验还包含了多 SNR 下的噪声鲁棒性测试，验证了系统在实际环境中的泛化能力。整个项目将信号与系统课程中的傅里叶变换、滤波器设计、采样定理、时频分析等核心理论，与现代深度学习技术紧密结合，具有较高的理论价值和工程实践意义。

## 一、问题的提出

### 1.1 研究背景

语音是人类最自然的交互方式。随着智能音箱、语音助手、车载语音控制等应用的普及，让机器准确理解人类语音命令成为一项重要的工程挑战。传统语音识别系统依赖隐马尔可夫模型 (HMM)、高斯混合模型 (GMM) 以及大量手工设计的声学特征（如 MFCC），流程复杂且需要领域专家深度参与。

近年来，深度学习特别是卷积神经网络 (CNN) 在图像识别领域的巨大成功启发了语音识别研究者：能否将语音信号转化为一种"图像"表示，再利用成熟的 CNN 架构进行端到端学习？

### 1.2 核心问题

本项目旨在解决以下三个核心问题：

1. **信号到图像的映射**：如何将一维时域语音信号转换为二维时频表示，使其既保留完整的语音信息，又适合 CNN 处理？
2. **端到端分类**：如何设计一个轻量级 CNN 模型，直接从频谱图中学习判别特征，无需手工特征工程？
3. **多语言扩展**：系统能否从单一语言（英文）扩展到多语言（中英双语）？如何处理不同语言之间数据量极度不均衡的问题？

### 1.3 项目对标

本项目在技术路线上对标了 EMG 肌电信号手势识别项目（CAE + CNN），两者的核心思想一致——将一维时序信号转化为二维表示后利用图像 CNN 进行分类。不同之处在于，语音信号的 2D 表示（梅尔频谱图）具有更明确的物理意义，且与"信号与系统"课程的理论关联更加紧密。

## 二、解决问题的原理与技术方案

### 2.1 系统总架构

```
│ .wav 文件 │    │ 预加重→STFT→Mel   │    │ 卷积网络  │    │ yes/no/.. │
└──────────┘    └──────────────────┘    └──────────┘    └──────────┘
                      ↑ 信号与系统核心          ↑ 深度学习
```

### 2.2 信号处理：从音频波形到梅尔频谱图

这是整个项目与"信号与系统"课程关联最紧密的部分。完整流水线包含六个步骤，每一步都有明确的数学基础。

#### 2.2.1 预加重 (Pre-emphasis)

语音信号在唇辐射过程中，高频分量的能量会以约 6 dB/oct 的速率衰减。预加重通过一个一阶高通 FIR 滤波器补偿这一衰减：

$$y[n] = x[n] - \alpha \cdot x[n-1], \quad \alpha = 0.97$$

其系统函数为 $H(z) = 1 - \alpha z^{-1}$，这是一个典型的一阶 FIR 高通滤波器。

#### 2.2.2 分帧与加窗 (Framing & Windowing)

语音信号具有短时平稳性——在 10-30 ms 的时间窗口内，信号的统计特性近似不变。因此，将长语音分割为重叠的短帧，对每一帧单独进行频谱分析：

$$\tilde{x}_m[n] = x[n + m \cdot H] \cdot w[n], \quad n = 0, 1, \ldots, N-1$$

本项目参数：帧长 $N = 400$ 点（25 ms @ 16 kHz），帧移 $H = 160$ 点（10 ms，60% 重叠）。

窗函数选用**汉明窗 (Hamming Window)**：

$$w[n] = 0.54 - 0.46 \cdot \cos\left(\frac{2\pi n}{N-1}\right)$$

加窗的目的是减少频谱泄漏——信号的突然截断会在频域产生旁瓣，汉明窗通过平滑两端衰减有效抑制这一效应。

#### 2.2.3 短时傅里叶变换 (STFT)

对每一帧加窗信号做 $K$ 点离散傅里叶变换（本项目 $K = 512$，补零至 2 的幂）：

$$X_m[k] = \sum_{n=0}^{N-1} \tilde{x}_m[n] \cdot e^{-j\frac{2\pi}{K}kn}, \quad k = 0, 1, \ldots, K-1$$

再计算功率谱：

$$P_m[k] = \frac{1}{K}|X_m[k]|^2$$

STFT 将一维时域信号 $\{x[n]\}$ 映射为二维时频矩阵 $\mathbf{P} \in \mathbb{R}^{(K/2+1) \times M}$，其中 $M$ 为帧数。

#### 2.2.4 梅尔滤波器组 (Mel Filterbank)

人耳对频率的感知是非线性的——低频区域的分辨率远高于高频区域。梅尔尺度 (Mel Scale) 通过一个对数映射模拟这一特性：

$$f_{\text{mel}} = 2595 \cdot \log_{10}\left(1 + \frac{f_{\text{Hz}}}{700}\right)$$

在梅尔域等间距放置 $B$ 个三角滤波器（本项目 $B = 128$），再映射回 Hz 域，得到低频密集、高频稀疏的非均匀滤波器组：

$$H_b[k] = \begin{cases}
0 & k < f_{b-1} \\
\frac{k - f_{b-1}}{f_b - f_{b-1}} & f_{b-1} \le k \le f_b \\
\frac{f_{b+1} - k}{f_{b+1} - f_b} & f_b < k \le f_{b+1} \\
0 & k > f_{b+1}
\end{cases}$$

将功率谱通过滤波器组得到各频带的能量：

$$S_m[b] = \sum_{k=0}^{K/2} P_m[k] \cdot H_b[k], \quad b = 1, 2, \ldots, 128$$

#### 2.2.5 对数压缩与归一化

人耳对响度的感知也近似对数关系（分贝尺度）。对数压缩使低能量区域（如辅音）的信息不被高能量区域（如元音）淹没：

$$S_m^{\text{dB}}[b] = 10 \cdot \log_{10}\left(S_m[b] + \varepsilon\right), \quad \varepsilon = 10^{-6}$$

最后进行 Z-score 归一化以稳定神经网络训练：

$$\hat{S}_m[b] = \frac{S_m^{\text{dB}}[b] - \mu}{\sigma}$$

**最终输出**：$\hat{\mathbf{S}} \in \mathbb{R}^{128 \times 98}$，即 128 个梅尔频带 × 98 个时间帧的"声音图像"，作为后续 CNN 的输入。

### 2.3 深度学习：2D 卷积神经网络

#### 2.3.1 为什么用 2D CNN？

CNN 的核心操作是**卷积**——这正是"信号与系统"中 LTI 系统的核心概念。在 CNN 中，卷积核是可学习的滤波器：

$$y[i, j] = \sum_{u=-k}^{k} \sum_{v=-k}^{k} w[u, v] \cdot x[i-u, j-v] + b$$

二维卷积天然适合处理具有局部相关性的二维数据。梅尔频谱图的横轴是时间、纵轴是频率，相邻的时频区域包含了语音的关键信息（如共振峰轨迹、谐波结构），这正是 2D CNN 擅长捕捉的模式。

#### 2.3.2 网络架构

本项目采用 3 个卷积块 + 全局平均池化 + 全连接分类器的设计：

```
Input: (1, 128, 98)  梅尔频谱图
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Block 1:
  Conv2D(1 → 32, 3×3) + BN + ReLU
  Conv2D(32 → 32, 3×3) + BN + ReLU
  MaxPool2D(2×2) + Dropout(0.2)
  → (32, 64, 49)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Block 2:
  Conv2D(32 → 64, 3×3) + BN + ReLU
  Conv2D(64 → 64, 3×3) + BN + ReLU
  MaxPool2D(2×2) + Dropout(0.3)
  → (64, 32, 24)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Block 3:
  Conv2D(64 → 128, 3×3) + BN + ReLU
  Conv2D(128 → 128, 3×3) + BN + ReLU
  GlobalAvgPool2D + Dropout(0.4)
  → (128,)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Classifier:
  Dense(128 → 256) + ReLU + Dropout(0.5)
  Dense(256 → num_classes) + Softmax
```

设计要点：
- **逐层增加通道数**（32 → 64 → 128）：底层学习局部纹理（基频、谐波），高层学习语义模式（音素、词汇）
- **Batch Normalization**：加速训练收敛，抑制内部协变量偏移
- **Global Average Pooling**：替代 Flatten 层，大幅减少参数量且具有平移不变性
- **Dropout**：训练时随机失活神经元，防止过拟合

#### 2.3.3 数据增强：SpecAugment

为提升模型的泛化能力和噪声鲁棒性，采用 Google Brain 提出的 SpecAugment 方法——直接在频谱图上进行增强操作：

- **频率遮盖 (Frequency Masking)**：随机遮挡连续的 Mel 频带（宽度 ≤ 27），模拟麦克风频响不平坦
- **时间遮盖 (Time Masking)**：随机遮挡连续的时间帧（宽度 ≤ 25，约 250 ms），模拟短时噪声

这些操作迫使模型从部分信息中学习完整的时频表示，效果显著且计算开销极小。

### 2.4 训练策略与优化

| 组件 | 配置 |
|:---|:---|
| 损失函数 | 交叉熵损失 (Cross-Entropy Loss) + 标签平滑 ($\epsilon = 0.1$) |
| 优化器 | AdamW ($\text{lr} = 10^{-3}$, $\beta_1=0.9$, $\beta_2=0.999$, weight decay = $10^{-4}$) |
| 学习率调度 | ReduceLROnPlateau (factor=0.5, patience=8) |
| 早停 | patience=20 (监控验证损失) |
| 批大小 | 8-32 (根据 GPU 显存调整) |
| 最大训练轮数 | 100 |

#### 2.4.1 类别加权损失（V3 特有）

在 V3 中英双语版本中，英文样本 (~54,000) 与中文样本 (~1,000) 之比约为 52:1。标准交叉熵损失会导致模型几乎完全忽略中文类别。为此在 V3 中引入了类别加权：

$$w_c = \frac{C}{n_c} \Big/ \sum_{c'=1}^{C} \frac{C}{n_{c'}}$$

其中 $C$ 为类别总数，$n_c$ 为类别 $c$ 的样本数。少数类（中文）获得约 50× 的权重，迫使模型在优化时给予同等重视。

## 三、实验/仿真验证

### 3.1 实验设置

#### 3.1.1 数据集

| 数据源 | 语言 | 样本数 | 来源 |
|:---|:---|:---|:---|
| Google Speech Commands v2 | 英文 | 77,454 | 2,618 名说话人，16 kHz，1 秒/条 |
| Microsoft Edge TTS 合成 | 中文 | 1,497 | 6 种声音 × 3 种语速，16 kHz，1 秒/条 |

数据划分：训练集 70%，验证集 15%，测试集 15%。

#### 3.1.2 信号处理参数

| 参数 | 值 | 理论依据 |
|:---|:---|:---|
| 采样率 $f_s$ | 16 kHz | Nyquist: $f_s \ge 2 \times 8\text{kHz}$ |
| 帧长 | 25 ms (400 点) | 语音短时平稳假设 (10-30 ms) |
| 帧移 | 10 ms (160 点) | 60% 重叠，保证时频谱连续性 |
| 窗函数 | Hamming | 减少频谱泄漏 |
| FFT 点数 | 512 | 2 的幂，补零至 512 |
| Mel 频带数 | 128 | 频率分辨率与计算量的平衡 |
| 频率范围 | 20 Hz - 8 kHz | 覆盖语音主要频段 |

### 3.2 V1：英文 20 类基准模型

V1 是只使用英文 GSC 数据的基准模型，共 20 个类别（10 个命令词 + 10 个数字）。

#### 3.2.1 训练过程

| 指标 | 数值 |
|:---|:---|
| 训练轮数 | 100 |
| 最佳轮次 | Epoch 96 |
| 最佳验证准确率 | 96.52% |
| 模型参数量 | 325,044 (~0.33M) |
| 训练耗时 | ~15 分钟 (GPU) |

训练过程平滑收敛，验证损失持续下降（2.88 → 0.87），验证准确率稳步上升（10.3% → 96.5%），未出现过拟合或早停触发。

#### 3.2.2 测试结果

| 指标 | 数值 |
|:---|:---|
| **测试准确率** | **96.49%** |
| 宏平均 F1 | 96.48% |
| 加权平均 F1 | 96.50% |

各类别表现均衡，最佳类别 stop (F1=98.02%)，最差类别 up (F1=91.56%)。这表明 CNN + 梅尔频谱方案在英文命令词识别上达到了很高的性能水平。

### 3.3 V2：中英双语 40 类扩展

在 V1 基础上，使用 Microsoft Edge TTS 合成 1,497 条中文语音（20 类），与英文数据混合构成 40 类数据集。中英文样本比约为 52:1。

#### 3.3.1 中文 TTS 数据合成

| 参数 | 值 |
|:---|:---|
| 合成引擎 | Microsoft Edge TTS (edge-tts) |
| 声音配置 | 6 种 (3 男声 + 3 女声) |
| 语速变化 | 0.85× / 1.0× / 1.15× |
| 实际成功量 | 1,497 条 (83.2%) |
| 每类样本 | 73-75 条 |

#### 3.3.2 双语测试结果

| 指标 | V1 (英文 20 类) | V2 (中英 40 类，未加权) |
|:---|:---:|:---:|
| 总体准确率 | 96.49% | **95.46%** |
| 英文准确率 | 96.49% | 95.8% |
| 中文准确率 | — | **76.1%** |
| 参数量 | 325,044 | 330,184 |

**分析**：类别数翻倍（20 → 40），总体准确率仅下降 1.03 个百分点，证明了 CNN 架构具有较强的容量和泛化能力。英文准确率基本持平（-0.7%），但中文仅 76.1%——这是由于训练数据中英文占 98%，中文仅占 2%，模型在优化过程中天然偏向英文类别。

### 3.4 V3：类别加权平衡训练

为解决 V2 中数据极度不均衡的问题，V3 在训练时引入了类别加权损失函数。中文类别的损失权重自动提升至约 50×，迫使模型在优化过程中给予中文同等的重视。

#### 3.4.1 V2 vs V3 对比

| 指标 | V2 (未加权) | V3 (加权) | 变化 |
|:---|:---:|:---:|:---:|
| 总体准确率 | 95.46% | **94.29%** | -1.17% |
| 英文准确率 | 95.8% | **94.5%** | -1.3% |
| **中文准确率** | **76.1%** | **82.9%** | **+6.8%** |
| Macro F1 | 0.839 | 0.822 | -0.017 |

**分析**：类别加权成功将中文准确率从 76.1% 提升至 82.9%（+6.8 个百分点），代价是英文准确率小幅下降 1.3 个百分点。这是典型的精度-公平性权衡——加权迫使模型"牺牲"部分英文性能来"学习"中文。

#### 3.4.2 V3 中文各类别详细表现

**表现优秀** (F1 ≥ 0.9)：zh_no, zh_zero, zh_stop, zh_two, zh_nine, zh_yes, zh_seven, zh_eight, zh_five

**表现一般** (0.5 ≤ F1 < 0.9)：zh_left, zh_go, zh_one, zh_three, zh_four, zh_six, zh_down

**表现较差** (F1 < 0.5)：zh_on (0.236), zh_up (0.0), zh_right (0.0)

表现较差的类别可能是 TTS 合成声音的清晰度不足或训练样本的多样性不够导致。总体而言，20 个中文类别中 16 个达到了可用的识别水平（F1 > 0.5），加权策略效果显著。

### 3.5 噪声鲁棒性测试

为验证系统在实际噪声环境中的性能，在测试集上叠加不同信噪比 (SNR) 的高斯白噪声进行测试。SNR 定义为：

$$\text{SNR(dB)} = 10 \cdot \log_{10}\left(\frac{P_{\text{signal}}}{P_{\text{noise}}}\right)$$

| SNR | V1 英文 | 说明 |
|:---|:---:|:---|
| Clean | 96.49% | 纯净语音 |
| 20 dB | ~95% | 微噪（安静办公室） |
| 15 dB | ~93% | 轻度噪声 |
| 10 dB | ~88% | 中度噪声 |
| 5 dB | ~75% | 较强噪声 |
| 0 dB | ~45% | 噪声功率 = 信号功率 |
| -5 dB | ~15% | 噪声盖过信号 |

结果表明系统在 SNR ≥ 10 dB（典型室内环境）下保持了较高的识别率（>88%），验证了基于梅尔频谱图的 CNN 方法对加性噪声具有一定的天然鲁棒性——这是因为频谱图中的语音谐波结构在中等噪声下仍可见，CNN 能从中提取有效特征。

**主要成果**：
1. 在 20 类英文命令词识别上达到 **96.49%** 的准确率，验证了梅尔频谱图 + CNN 方案的有效性
2. 成功扩展至 40 类中英双语识别，中文准确率达 **82.9%**（加权版本）
3. 通过类别加权损失函数有效缓解了 52:1 的极度数据不均衡问题
4. 进行了多 SNR 下的噪声鲁棒性测试，系统在 ≥10 dB 环境下保持 88%+ 准确率

**信号与系统理论应用**：本项目将课程核心理论——傅里叶变换（FFT）、短时傅里叶变换（STFT）、采样定理（Nyquist）、滤波器设计（Mel 滤波器组、汉明窗）、信噪比（SNR）等——直接应用于工程实践，实现了理论与应用的有机结合。

**未来工作**：增加中文 TTS 数据量以从根本上解决不均衡问题、引入 Conformer/Transformer 架构进一步提升性能、探索自监督预训练（如 wav2vec 2.0）在少样本语音识别中的应用。
