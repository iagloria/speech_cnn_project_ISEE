## 2.10 V1/V3 附：传统方法对比验证

| 方法 | 总体 Acc | Macro F1 | EN | ZH | 训练 |
|:---|:---:|:---:|:---:|:---:|:---:|
| **CNN (V3)** | **94.29%** | 82.17% | **94.5%** | 82.9% | 15 min |
| SVM (RBF) | 48.88% | 46.08% | 49.1% | 37.8% | 2 s |
| Random Forest | 69.63% | **83.57%** | 69.1% | **98.2%** | 31 s |
| KNN (k=5) | 49.97% | 40.55% | 49.3% | 83.8% | <1 s |

核心发现:
  1. CNN 是唯一中英均衡的方法 (EN 94.5% + ZH 82.9%), 传统方法两极分化严重
  2. RF 中文 98.2% 但英文仅 69.1%: TTS 合成语音模式单一(6种声音×3语速),
     RF 靠记忆特定频带能量分布即可完美分类; 英文 2,618 人自然语音多样性大,
     RF 无法泛化 → 暴露了传统方法依赖手工特征、缺乏平移不变性的根本缺陷
  3. SVM/KNN 在中英文上均表现差: 经 PCA 降至 256 维后时频局部信息大量丢失
  4. CNN 的 2D 卷积核天然提取时频局部模式 (共振峰轨迹、谐波结构),
     且 MaxPool 提供平移不变性 (同一单词不同语速/音调), 这是传统方法不具备的

结论: CNN 通过端到端学习自动发现了对语音分类最优的时频特征表示,
      无需 PCA 降维或手工特征工程, 在中英跨语言场景下具有最强的泛化能力。

## 2.13 V4 vs V3 对比分析 (关键洞察)

中文不升反降的原因:
  1. TTS 过拟合: --num_repeats 30 虽然生成了 9,000 条中文,
     但全部来自同样的 6 种声音 × 3 种语速 = 18 种模式。
     过采样让模型在每个 batch 中反复看到这些重复模式,
     导致对 TTS 训练集的记忆而非泛化。
  2. 测试集变化: V4 测试集中文从 222 条扩到 1,462 条,
     包含更多样的 TTS 声音组合, 评估更严格。
  3. 传统方法验证: RF/KNN 在 V4 中文测试集上达到 100%,
     证明确实存在 TTS 模式泄露——传统方法靠死记硬背即可满分,
     CNN 反而因为更强的泛化约束而表现更差。

结论: 9,000 条 TTS 数据虽然量大了, 但多样性 (18 种声音组合)
      并未实质增加。真正的提升需要真人录音或更多样化的 TTS 声音。

## 2.16 泛化诊断: 真人中文识别差

经 gui_test.py 反复测试 + TTA 优化后, 中文识别效果仍不理想。
使用 compare_spectrograms.py 对比真人录音与 TTS 同词频谱:
  - 发现真人频谱与 TTS 频谱存在显著分布差异 (MSE大, 相关系数低)
  - 根因: TTS 训练数据(6声×3速=18种模式)过于"干净整洁",
    与真人语音的声学特征(房间混响/口音/自然音高变化)不匹配
  - CNN 学到的中文特征是"TTS 特征"而非"中文语音特征"

解决方案: 真人中文录制(每类10-20条) → V4 迁移学习微调
  python train.py --data_dir ../data/processed_mixed_v4 --resume \
    ../models/v2_bilingual_40/bilingual_v4/best_model.pth --epochs 20 --lr 1e-4

## 2.17 推理优化: 测试时增强 (TTA)

gui_test.py 集成了 TTA 推理:
  - 一次录音 → 5个变体(±1半音/×0.9×1.1语速+原始)
  - 5次推理 → 平均logits → 最终预测
  - 预处理: 去直流+激进去静音+预加重+RMS归一化+峰值限幅

## 2.7 踩坑记录: 中文"全错"乌龙

现象: 多次评估显示中文 20 类 Support=0 / F1=0.000。
根因: evaluate.py 默认加载 data/processed (V1 20类数据),
      测试集中根本没有中文标签 (0-19), 导致中文 (20-39) Support=0。
解决过程:
  1. 最初 evaluate.py 硬编码 data_dir='../data/processed', V2 评估需手动加 --data_dir
  2. 改进为读模型 config.json 的 NUM_CLASSES 自动选择数据集
  3. 修复后 bilingual_v1 的中文真实准确率 = 76.1%, 并非 0%

三、信号处理参数 (报告理论基础)
======================================================================

| 参数 | 值 | 理论关联 |
|------|-----|----------|
| 采样率 | 16kHz | 满足 Nyquist 定理 (f_s ≥ 2×f_max=8kHz) |
| 预加重系数 | α=0.97 | 一阶高通 FIR 滤波器 H(z)=1-αz⁻¹ |
| 帧长 | 25ms (400点) | 语音短时平稳假设 (10-30ms) |
| 帧移 | 10ms (160点) | 帧间 60% 重叠, 平滑时频谱 |
| 窗函数 | Hamming | w[n]=0.54-0.46cos(2πn/N), 减少频谱泄漏 |
| FFT 点数 | 512 | 2的幂, 补零到 512 |
| Mel 频带数 | 128 | 三角滤波器组, mel(f)=2595·log₁₀(1+f/700) |
| 频率范围 | 20Hz-8kHz | 覆盖语音主要频段 |
| dB 压缩 | 10·log₁₀(S+ε) | 模仿人耳对数响度感知 |
| 归一化 | Z-score | (x-μ)/σ, 稳定训练 |
